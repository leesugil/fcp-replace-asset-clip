# fcp-replace-asset-clip
The built-in FCP asset-clip replacement is not ideal when there's a .01 difference in the imported fps value (even if you assure that the original asset and the replacing asset are essentially the same). This happens when you work with media with a variable frame rate (VFR). This tool safely replaces (possibly too many) asset-clips with vfr sources in a Project Timeline.

This fixes the 58.82 fps curse in FCP.
